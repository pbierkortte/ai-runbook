# agents-collate.sh

**Source:** `script/agents-collate.sh`

---

## What It Does

`agents-collate.sh` finds every `AGENTS.md` in the repository and rewrites it.

For each `AGENTS.md` found, the script:

1. Changes into the directory that contains it.
2. Overwrites `AGENTS.md` with a `<!-- THIS FILE IS AUTOMATICALLY GENERATED DO NOT EDIT -->` header.
3. Scans every sibling `.md` file in that directory (excluding `AGENTS.md` itself, case-insensitive).
4. Extracts markdown headings from each file using `awk`, skipping code fences.
5. Appends a heading-based catalog entry for each file.

The result is a flat, heading-only index that an AI agent can read to discover what documentation exists, without loading every full document into context.

---

## How to Run

```bash
bash script/agents-collate.sh
```

Run from any directory. The script resolves the repo root from its own location.

---

## AGENTS_INCLUDE_DIR

`AGENTS_INCLUDE_DIR` is an optional environment variable.

When set to an existing directory path, the script appends headings from every `.md` file in that directory to the **root-level `AGENTS.md` only**.

Subdirectory `AGENTS.md` files are not affected by `AGENTS_INCLUDE_DIR`.

If `AGENTS_INCLUDE_DIR` is unset, empty, or points to a path that does not exist, the script runs unchanged.

---

## bootstrap

**Source:** `script/bootstrap`

`bootstrap` exports `AGENTS_INCLUDE_DIR` with a default value:

```bash
export AGENTS_INCLUDE_DIR="${AGENTS_INCLUDE_DIR:-$HOME/ai-runbook}"
```

If `AGENTS_INCLUDE_DIR` is already set in the environment, `bootstrap` leaves it unchanged. If it is not set, `bootstrap` sets it to `$HOME/ai-runbook`.

Source `bootstrap` before running `agents-collate.sh` in a Codespace:

```bash
source script/bootstrap
bash script/agents-collate.sh
```

---

## The Problem It Solves

A developer maintains personal AI agent rules in a dotfiles repo at `$HOME/ai-runbook`. Those files are not siblings of any project's `AGENTS.md`. Without `AGENTS_INCLUDE_DIR`, `agents-collate.sh` never reads them. The root `AGENTS.md` is generated without the personal rules. The rules exist. The agents never see them.

`AGENTS_INCLUDE_DIR` adds one external directory to the root catalog. `bootstrap` sets that variable automatically in Codespaces so no manual step is required per project.

The target project gains no new files, no symlinks, and no `.gitignore` changes.

---

## Behavior Summary

| Condition | Result |
|---|---|
| `AGENTS_INCLUDE_DIR` unset | Script runs normally; no external files included |
| `AGENTS_INCLUDE_DIR` set, path does not exist | Script runs normally; no external files included |
| `AGENTS_INCLUDE_DIR` set, path exists | Root `AGENTS.md` receives headings from `*.md` files in that directory |
| Subdirectory `AGENTS.md` | Never receives `AGENTS_INCLUDE_DIR` content |
| Running collate twice | Output is identical; the script is idempotent |
